01 [0-16.628] 用 Codex 或者 Claude Code 干点稍微复杂的活，经常是好几步：先改代码，再跑测试，最后让它提交。麻烦的是，每一步你都得守着——发一条，等它答完，再发下一条。
02 [16.628-32.488] 你可能会想，那我一次把几步全发过去不就完了？可那样会直接打断它正在写的回答，全乱。后来我用上一个小工具，叫 prompt-flow，它的 Flow 模式就是专门解决这个的。
03 [32.488-47.054] 做法很聪明：你把这几步提示词按顺序排好，按 Ctrl、Alt、P 选中这条 Flow。它会等当前这条回答彻底结束，再自动发下一条，全程不打断。
04 [47.054-62.826] 说再多不如直接看。你看，选一次它就自己往下跑了。这时候你完全可以走开，去刷个网页、切到别的窗口，只要别把它关掉，它就一直接着发。这是 Codex。
05 [62.826-74.458] 换成 Claude Code 也一样。一条 Flow 设好，多步任务自己跑完，你回来直接看结果就行。值得试试。
